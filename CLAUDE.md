# CLAUDE.md

## Правила работы с проектом

### Главный принцип
- **Не создавать дрифт.** Не хардкодить в документации и коде конкретные числа, счётчики, списки, которые устареют после правок. Если значение меняется — оно должно вычисляться или браться из единого источника. Это касается всего: кода, тестов, документации, комментариев.

### Контент
- **content.js — единственный источник правды.** Весь текст, даты, ссылки — только там. Не хардкодить в HTML, JS или generate-cv.js.
- **Русский и английский синхронизированы.** При любом изменении в `CONTENT.ru` — обязательно обновить `CONTENT.en`. Количество items в каждой секции должно совпадать (тест это проверяет).
- **Перевод на английский — литературный, не дословный.** Сохранять смысл и интонацию, но сложные русские конструкции упрощать. Английский текст должен звучать естественно для носителя, а не калькой с русского. Лучше перефразировать, чем переводить в лоб.
- **Технические enum-поля не переводятся.** Поля с ограниченным набором допустимых значений (`status: "hot"/"active"/"exploring"`, `icon: "tg"/"gh"/...`) одинаковы в RU и EN. Переводить надо только человеко-читаемый текст (`t`, `d`, `label`).
- **Даты и числа вычисляются, а не хардкодятся.** Возраст, стаж — из `meta.birth`, `meta.start_it`, `meta.start_devops`. Не писать "13 лет" в тексте.
- **`meta.last_updated` обновлять при правке контента.** Тест напомнит если дата старше 3 месяцев.

### Код
- **НИКОГДА не коммитить без явного разрешения пользователя.** «Разрешение» — это слова вроде «коммить», «можно коммитить», «ок, коммит». «Отлично», «хорошо», «погнали», «разгоняем» — это про работу, не про коммит. Если не уверен — спроси ещё раз и жди явного ОК. Даже после прохождения тестов, даже когда диффа очевидно готов, даже при auto-mode.
- **Одна фича — один коммит.** Не смешивать несвязанные изменения.
- **Рефакторинг перед коммитом.** Проверить дифф, убрать мусор, мёртвый код, лишние комментарии.
- **Тесты на новые фичи.** Если добавляешь секцию, поле или файл — добавь проверку в test.js.
- **`_lang` — глобальное состояние языка.** Не прокидывать lang параметром через всю цепочку, читать `_lang`.
- **escapeHTML() для пользовательского контента** в HTML-атрибутах и опасных контекстах.
- **Dynamic CSS-классы → `data-*` атрибуты, а не template-literal.** `class="foo-${state}"` прячет все варианты от dead-code-тестов (подстрока не находится в JS). Использовать `class="foo" data-state="${state}"` + CSS-селекторы `[data-state="value"]` — так все возможные значения явно перечислены в CSS и видны тестам.

### Стиль
- **Mobile-first.** Базовые стили — мобильные, расширяем через `min-width`.
- **CSS-переменные из base.css.** Не хардкодить цвета и шрифты.
- **Нет `!important`** кроме `prefers-reduced-motion` (тест это проверяет).
- **Preview перед правкой.** Любое визуальное изменение — сначала preview в браузере (Playwright + CSS-inject на live-страницу), потом применять в файлах. Не «первое что придёт» — готовить 2–3 варианта и показывать бок-о-бок. Скорее сэкономит раунды «показал — не зашло — переделал».
- **Ритм секций важнее отдельной секции.** Страница при скролле — это поток: яркое → тихое → яркое → тихое. Три ярких подряд = визуальная инфляция. Pattern break в секциях не должен повторять voice hero (terminal-язык — только там, иначе hero обесценивается).
- **Transitions синхронны, без layout-shift.** Все hover-свойства одного компонента — одинаковая длительность и easing. Разные durations для разных свойств = «разъезжание» анимации. `padding-left/margin/size` на hover запрещены — создают reflow; оставлять `color`, `background`, `transform`, `opacity`.
- **Reveal через `@keyframes animation`, hover через `transition`.** `transition-delay` у родителя (например, у `.stagger > *:nth-child(N)`) протекает на ВСЕ transitions потомков — в результате hover на вложенных элементах «тормозит» пропорционально nth-child. Stagger-reveal делать через `@keyframes` + `animation-delay` — animation one-shot, другие transitions не трогает.
- **Revert-friendly блоки для экспериментальных фич.** Крупные опциональные правки (polaroid-wall, big-typography и т.п.) оформлять как самостоятельный блок в `components.css` с маркером `═════════` в начале/конце. Один блок = одна фича = откат через удаление. Не вплетать экспериментальные стили поверх базовых селекторов — держать их изолированно на специфичном префиксе (`.sect-human .human-card` поверх базового `.human-card`).

### Изображения
- **Только WebP** в assets/. Формат: `cwebp -q 82 -resize 800 0 input -o output.webp`.
- **.gitignore блокирует jpg/png/PNG** в assets/ — сырые файлы не попадут в репу.
- **Lazy loading** на всех изображениях (`loading="lazy"`).

### PDF
- **`node generate-cv.js`** — русский PDF. **`node generate-cv.js --en`** — английский.
- **Pre-commit hook** автоматически перегенерирует оба PDF при изменении content.js или generate-cv.js.
- **Заголовки секций в PDF** берутся из content.js, не хардкодятся.

### Тесты
- **тесты** запускаются автоматически при каждом коммите.
- **Коммит блокируется** при провале тестов.
- **`node test.js`** — запуск вручную. **`SKIP_PDF_GEN=1 node test.js`** — без перегенерации PDF.

### Git
- **Pre-commit hook** (`.githooks/pre-commit`): тесты + PDF + sitemap lastmod.
- **`core.hooksPath = .githooks/`** — хуки в репе, версионируются.
- **Не пушить без тестов.**
